# Romo AI Engineering Principles

## Principle 1
Every folder has one responsibility.

## Principle 2
Every file should be explainable in less than 30 seconds.

## Principle 3
Simple beats clever.

## Principle 4
Code for the next engineer.

## Principle 5
Do not build features that do not create value.

## Principle 6
Document important decisions.

## Principle 7
Never depend directly on one AI provider.

## Principle 8
Every component should be reusable or intentionally isolated.

## Principle 9
No temporary names in production code.

Do not use:
- temp
- final
- new
- test2
- copy
- old

## Principle 10
If it does not save time, create value, improve decisions or reduce complexity, it does not belong in Romo AI.
